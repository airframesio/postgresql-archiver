Accessibility

All components meet WCAG 2.1 Level AA standards. This includes proper color contrast, keyboard navigation, focus indicators, and screen reader support.

Accessibility Checklist

  • ✓ Color contrast ratio ≥ 4.5:1 for normal text
  • ✓ Touch targets minimum 44x44px on mobile
  • ✓ Visible focus indicators on all interactive elements
  • ✓ Keyboard navigation support
  • ✓ ARIA labels and roles
  • ✓ Screen reader announcements
  • ✓ Skip to content links
  • ✓ Semantic HTML structure

Best Practices

Follow these guidelines to maintain consistency and quality across the application.

Do's and Don'ts

✓ Do

  • Use design tokens for all values
  • Test with keyboard navigation
  • Provide alternative text for images
  • Use semantic HTML elements
  • Test in both light and dark modes

✗ Don't

  • Use hard-coded colors or spacing
  • Create elements without focus states
  • Rely solely on color to convey meaning
  • Use divs for interactive elements
  • Forget to test mobile responsiveness
